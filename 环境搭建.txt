一、基础软件安装（如已有基础环境跳至第二步）
----------------------------------------------------------------

【1】Python 3.11+(推荐 3.13.12 不要高于 3.13版本 否则后续步骤可能出错)
    下载地址：https://www.python.org/downloads/
    安装时勾选 "Add Python to PATH"
    验证：
        python --version
        pip --version

【2】Node.js 18+（前端必须）
    下载地址：https://nodejs.org/（选 LTS 版本）
    验证：
        node --version
        npm --version

【3】MySQL 8.0+
    下载地址：https://dev.mysql.com/downloads/mysql/
    推荐安装方式：MySQL Installer（含 MySQL Server + Workbench）
    安装时记住 root 密码，后续配置要用。
    验证：
        mysql -u root -p
        输入密码后看到 mysql> 提示即成功

【4】Git（可选，用于拉取代码）
    下载地址：https://git-scm.com/downloads/win

【5】Microsoft Visual C++ 编译工具（chromadb依赖下载时需要）
    下载地址：https://visualstudio.microsoft.com/zh-hans/visual-cpp-build-tools/
    安装时，确保勾选以下组件：
    MSVC v143 - VS 2022 C++ x64/x86 build tools
    Windows 10/11 SDK
    安装完成后，重启电脑

【6】 安装语音文件转码工具 （可选，需要上传语音文件的话就安装）
  ffmpeg基础版 
  https://wwt.lanzouo.com/i8Boq1rnfbcf?password=49si 密码 49si 
  解压之后将路径配置环境变量
  示例： E:\ffmpeg\ffmpeg-6.1.1-essentials_build\bin
 验证： ffmpeg -version

二、Python 虚拟环境创建

    # 进入项目目录

三、安装 Python 依赖（基于venv虚拟环境）
----------------------------------------------------------------

    #1. 升级 pip
    pip install --upgrade pip
    # 如果报错则强制安装
    python -m pip install --upgrade --ignore-installed pip

    # 2.安装所有依赖（需要几分钟，包含大型库如 sentence-transformers）
    pip install -r requirements.txt

  ⚠️ 注：如果 pip install -r requirements.txt 命令执行很慢
      切换国内镜像：
       pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple


    ⚠️ 注意：camelot-py[cv] 依赖 opencv，若安装失败可跳过：
        pip install -r requirements.txt --ignore-requires-python
    或单独移除 requirements.txt 中的 camelot-py 那行后再安装。

五、配置环境变量（.env 文件）
----------------------------------------------------------------

    项目目录下已有 .env 文件，根据实际情况修改：

    # 用记事本或 VS Code 打开
     找到\rag-system\.env 文件

    必须修改的配置项：

    ┌─────────────────────────────────────────────────────┐
    │  MYSQL_PASSWORD=你的MySQL root密码                   │
    │  MYSQL_USER=root                                    │
    │  MYSQL_DATABASE=rag_system                          │
    └─────────────────────────────────────────────────────┘

    按需修改（填写你要用的大模型 API Key   也可页面配置）：

    ┌─────────────────────────────────────────────────────┐
    │  DEFAULT_LLM_PROVIDER=deepssek  （选一个）             │
    │                                                     │
    │  # 使用 OpenAI                                      │
    │  OPENAI_API_KEY=sk-xxxx                             │
    │                                                     │
    │  # 使用 DeepSeek（推荐，便宜）                        │
    │  DEEPSEEK_API_KEY=sk-xxxx                           │
    │                                                     │
    │  # 使用通义千问                                       │
    │  DASHSCOPE_API_KEY=sk-xxxx                          │
    │                                                     │
    │  # 使用本地 Ollama（免费，见下方第六步）                │
    │  DEFAULT_LLM_PROVIDER=ollama                        │
    └─────────────────────────────────────────────────────┘

    Embedding 模型（默认本地，首次运行自动下载）：
    ┌─────────────────────────────────────────────────────┐
    │
    │  EMBEDDING_PROVIDER=local                           │
    │  EMBEDDING_MODEL=BAAI/bge-m3                        │
    │  （模型约 1.1GB，首次运行自动下载到本地）          │
    │  在 .env 里配置环境变量：
    │   HF_HOME= xxxx(下载到本地路径)
    │
    └─────────────────────────────────────────────────────┘

六、【可选】安装 Ollama（使用本地大模型，无需 API Key）
----------------------------------------------------------------

    下载地址：https://ollama.com/download

    安装后拉取模型：
         ollama pull deepseek-r1:7b   （DeepSeek 7B，约 4.7GB）
    启动 Ollama 服务：
        ollama serve
        （默认运行在 http://localhost:11434）

    在 .env 中设置：
        DEFAULT_LLM_PROVIDER=ollama
        OLLAMA_MODEL= deepseek-r1:7b

九、常见问题排查
----------------------------------------------------------------D

Q2：MySQL 连接失败 (Can't connect to MySQL server)
    A：检查 MySQL 服务是否启动
       Windows：Win+R → services.msc → 找到 MySQL80 → 启动

Q3：首次启动后端很慢（卡在启动阶段）
    A：正在下载 BAAI/bge-m3 Embedding 模型（约 1.1GB），等待即可
       下载完成后后续启动不需要重新下载

Q4：端口 8000 被占用
    A：修改启动命令：uvicorn main:app --reload --port 8001
       同时修改 frontend/vite.config.js 中 proxy target 为 8001

Q5：camelot-py 安装失败
    A：删除 requirements.txt 中 camelot-py[cv] 那行，该库仅用于
       PDF 表格高精度提取，不影响其他功能

Q6：sentence-transformers 模型下载失败（网络问题）
    A：设置镜像后重试：
       在 .env 中改为云端 Embedding：
           EMBEDDING_PROVIDER=openai
           OPENAI_API_KEY=sk-xxxx

Q7：前端页面空白/接口 404
    A：确认后端已正常启动，vite.config.js proxy 配置正确
